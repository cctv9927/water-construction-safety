# 水利建设工地质量安全监管系统 - 项目概述

## 项目背景

**项目名称**：水利建设工地质量安全监管系统（WaterConstruction-Safety）

**目标**：对水利建设工地实现全面的质量安全智能化监管

**交付范围**：纯软件平台，不包含硬件和传感器

**智能化程度**：高度智能化（感知数据自动采集、AI 自动分析、全自动闭环处理）

---

## 三层架构

### 第一层：感知层（软件部分）

#### 1.1 数字传感器数据采集
- 温度、压力、震动、位移、流量、风速、降雨量等传感器
- 接入方式：MQTT 协议采集传感器数据
- 数据格式：JSON，统一上报到 IoT Hub

#### 1.2 视频摄像头
- 支持 RTSP 拉流（主流摄像头协议）
- 视频流处理：FFmpeg 截帧、推流
- 传输：WebSocket 实时推送 + HLS 录像回放

#### 1.3 无人机
- DJI SDK 集成（通过大疆开放平台）
- 视频流回传：RTMP → 转 WebRTC
- 航线规划与控制指令下发

---

### 第二层：人工智能平台

#### 2.1 图片智慧识别
- **识别目标**：人（安全帽/未戴帽）、机（施工机械）、料（建材堆放）、环（环境安全隐患）、法（违规操作）
- **模型**：YOLOv8（目标检测）+ 图像分类
- **推理**：ONNX Runtime，边缘+云端双部署
- **输出**：检测框 + 类别 + 置信度 + 位置坐标

#### 2.2 智慧 3D 建模
- 基于施工现场照片重建 3D 模型
- 使用工具：Photogrammetry（COLMAP/OpenMVS）
- 输出：GLTF 格式，可嵌入电子沙盘

#### 2.3 视频处理
- 实时流目标追踪（DeepSort）
- 视频浓缩（将长时间视频浓缩为关键片段）
- 视频质量诊断（遮挡、遮挡、角度异常）

#### 2.4 语音处理
- 语音识别（Whisper 中文模型）
- 语音播报（告警通知）
- 语音指令控制（关键词触发）

#### 2.5 多智能体协同
- 感知层数据 → AI 分析 Agent → 决策 Agent → 执行 Agent
- 异常事件自动分级（P0/P1/P2）
- 多模态融合：视频+传感器+语音 联合判断

---

### 第三层：应用层

#### 3.1 电子沙盘展示平台
- 3D GIS 地图（CesiumJS / MapboxGL）
- 实时监控画面嵌入
- AI 检测结果叠加标注
- 传感器数据可视化（热力图、数值仪表盘）

#### 3.2 问题闭环管理平台
- 问题登记 → 派发 → 处理 → 复核 → 归档
- 状态流转：待处理 → 处理中 → 已完成 → 已复核
- 关联：问题 ↔ 视频片段 ↔ 传感器数据 ↔ 3D 标注

#### 3.3 问题提醒提示平台
- 实时告警（WebSocket 推送）
- 多渠道通知：APP 推送 / 短信 / 邮件
- 告警分级：P0（立即处理）/ P1（当天处理）/ P2（计划处理）

#### 3.4 质量安全专家系统
- 知识库问答（基于 RAG）
- 辅助办公（检查表自动生成、工作日志）
- 各类表格智慧填报（检查记录表、整改通知书、验收报告）
- 智能推荐（基于历史问题推荐处理方案）

---

## 技术选型

| 模块 | 技术栈 |
|------|--------|
| 后端框架 | FastAPI（异步、高性能） |
| 数据库 | PostgreSQL（主数据）+ TimescaleDB（时序传感器数据）|
| 消息队列 | Redis Stream（实时告警）|
| 视频流 | FFmpeg + WebRTC |
| AI 推理 | ONNX Runtime + PyTorch |
| 前端框架 | React + TypeScript |
| 3D 地图 | CesiumJS |
| 视频播放 | 原生 MSE + HLS.js |
| 无人机 | DJI Web SDK |
| 知识库 | Milvus（向量数据库）+ RAG |
| 部署 | Docker + Kubernetes |
| CI/CD | GitHub Actions |
