## 2. 模块职责说明

### 2.1 感知层（Perception Layer）

| 模块 | 技术栈 | 职责 | 数据出口 |
|------|--------|------|---------|
| **传感器设备** | 温度/压力/位移/振动等 | 采集工地物理参数 | MQTT → ThingsBoard |
| **摄像头** | RTSP 协议 | 视频画面采集 | RTSP → EasyDarwin |
| **无人机** | DJI SDK | 航拍巡查、航线巡检 | RTMP → EasyDarwin |
| **ThingsBoard** | Java/Spring Boot | IoT 平台（设备管理、规则引擎、时序数据）| REST/MQTT |
| **EasyDarwin** | Go | 流媒体服务器（RTMP 推流、HTTP-FLV 拉流）| HTTP-FLV |

---

### 2.2 AI 平台层（AI Platform Layer）

| 模块 | 技术栈 | 职责 | 输入 | 输出 |
|------|--------|------|------|------|
| **AI-Vision** | YOLOv8 + ONNX Runtime | 图像目标检测（安全帽、越界、烟火等）| 视频帧 | 检测结果 JSON |
| **AI-Voice** | Whisper + EdgeTTS | 语音转文字、TTS 语音播报 | 音频流 | 文本/音频 |
| **Video-Streamer** | Python/OpenCV | 视频流处理、帧提取、ByteTrack 多目标追踪 | RTSP 流 | 追踪轨迹 |
| **Sensor-Collector** | Python/paho-mqtt | 从 ThingsBoard 订阅传感器数据，转发到 Redis | MQTT | Redis Stream |
| **AI-Coordinator** | Python | 多智能体协调调度、RAG 问答编排、LLM 调用 | Redis/ HTTP | JSON 响应 |

---

### 2.3 应用层（Application Layer）

| 模块 | 技术栈 | 职责 |
|------|--------|------|
| **Backend API** | FastAPI + SQLAlchemy | REST API 聚合、统一认证、数据持久化、WebSocket 推送 |
| **Gateway** | Python | 统一接入网关（协议转换、限流、鉴权）|
| **知识库（RAG）** | Milvus + LangChain | 文档向量化存储、语义检索、LLM 生成 |

---

### 2.4 用户层（User Layer）

| 模块 | 技术栈 | 职责 |
|------|--------|------|
| **地图沙盘前端** | React + CesiumJS + Ant Design | 三维工地地图、传感器叠加、视频点位 |
| **闭环管理前端** | React + Ant Design Pro | 告警全流程管理、处置工作流、统计报表 |
| **专家系统前端** | React + RAG UI | 知识问答、表格生成、事故分析 |

## 3. 数据流向

### 3.1 传感器数据流

```
传感器设备 ──MQTT──► ThingsBoard ──Webhook/HTTP──► Sensor-Collector
                                                          │
                                                          ▼
                                                    Redis Stream
                                                          │
                                                          ▼
                                               AI-Coordinator（分析）
                                                          │
                                                          ▼
                                              Backend API（存储 + 推送）
                                                          │
                                                          ▼
                                           WebSocket / SSE ──► 前端展示
```

### 3.2 视频数据流

```
摄像头 ──RTSP──► EasyDarwin ──RTMP──► Video-Streamer
                              │
                              ▼
                       AI-Vision（YOLOv8 检测）
                              │
                              ▼
                       AI-Coordinator（告警判断）
                              │
                              ▼
                      Backend API（告警入库）
                              │
                              ▼
               飞书 Webhook（告警推送）/ WebSocket（前端）
```

### 3.3 RAG 问答数据流

```
用户问题 ──► Backend API ──► AI-Coordinator ──► RAG Pipeline
                                                    │
                              ┌─────────────────────┘
                              ▼
                       Milvus（向量检索）
                              │
                              ▼
                       Top-K 相关文档片段
                              │
                              ▼
                       LLM（基于上下文生成答案）
                              │
                              ▼
                       返回答案 + 引用来源
```

## 4. 技术选型理由

| 技术 | 选型理由 |
|------|---------|
| **FastAPI** | 异步高性能、自动 OpenAPI 文档、类型安全 |
| **TimescaleDB** | 时序数据压缩、自动分区、SQL 兼容 |
| **Redis Stream** | 高吞吐消息队列、支持消费组、持久化 |
| **Milvus** | 开源向量数据库、支持 ANN 检索、水平扩展 |
| **YOLOv8 + ONNX** | 高精度目标检测、跨平台推理优化 |
| **Whisper** | 开源语音识别、多语言支持、精度高 |
| **CesiumJS** | 开源三维地球引擎、WebGL 渲染、支持倾斜摄影 |
| **ThingsBoard** | 开源 IoT 平台、设备管理成熟、社区活跃 |
| **Docker Compose** | 快速部署、版本锁定、环境隔离 |

## 5. 安全设计

| 安全措施 | 实现方式 |
|---------|---------|
| **认证授权** | JWT Bearer Token，支持 Token 刷新 |
| **密码存储** | bcrypt 哈希（cost factor 12）|
| **API 限流** | 全局限流（Redis 实现滑动窗口）|
| **CORS** | 生产环境白名单域名配置 |
| **SQL 注入** | SQLAlchemy ORM 参数化查询 |
| **审计日志** | 所有写操作记录操作日志 |
| **敏感配置** | 环境变量注入，不硬编码密钥 |
